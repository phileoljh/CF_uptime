# HihiMonitor — CloudFlare Workers 多網站監控系統

HihiMonitor 是一個基於 Cloudflare 邊緣網路架構所建立的輕量級、高可用性網站監控系統。透過 Cloudflare Workers 執行監控邏輯，D1 作為關聯式資料庫儲存紀錄，並結合 Zero Trust 確保儀表板安全。本專案可達成長期且無需任何伺服器成本的網站可用性監控。

## 系統特色

- **零成本運行**：完全建立於 Cloudflare Free Tier (免費層) 之內。
- **防抖機制 (Debounce)**：連續兩次檢查失敗才會發送「故障」通知 (ADD)，連續兩次成功才會發送「恢復」通知 (DAA)，有效減少暫時性網路波動導致的誤報。
- **持續故障報警 (DDDDDD)**：針對超過 8 小時無法恢復的網站，系統會再次發出警告，確保問題沒有被遺忘。
- **無伺服器架構**：無需安裝本地端開發環境 (Node.js 等)，完全可以在 Cloudflare Dashboard 網頁介面完成包含部署在內的所有維護作業。
- **Zero Trust 儀表板**：可限制特定信箱才能存取觀看監控儀表板。

## Cloudflare 免費層限制 (Free Tier Limits)

本專案使用到了兩種主要的 Cloudflare 服務：Workers 及 D1 資料庫。以下是 Cloudflare 官方明訂的免費層額度：

### 1. Cloudflare Workers 免費限制
- **請求數 (Requests)**：100,000 次 / 每天
- **CPU 時間 (CPU Time)**：10 毫秒 (ms) / 每請求 (大部分網路請求的等待時間不計入 CPU 時間內)
- **Worker 腳本數量上限**：每個帳號最多可建立 **100 個** Worker
- **Cron Trigger 上限**：每個 Worker 腳本最多可綁定 **3 個** Cron Triggers

### 2. Cloudflare D1 (SQLite) 免費限制
- **資料讀取 (Row Reads)**：5,000,000 筆 / 每天
- **資料寫入 (Row Writes)**：100,000 筆 / 每天 (包含 INSERT, UPDATE, DELETE)
- **總儲存空間 (Storage)**：5 GB

## 本實作資源消耗推估

根據本系統預設配置（設定 **3 個受監控網站**、設定每 **5 分鐘探測一次**、日誌保留 **90 天**）進行資源計算：

### 1. Workers 請求次數分析
- 每 5 分鐘執行一次 Cron 健康檢查 $\rightarrow$ **288 次 / 天**
- 每天凌晨 03:00 執行一次 Cron 舊資料清理 $\rightarrow$ **1 次 / 天**
- 總自動請求：**289 次 / 天**
- **結論**：加上人工偶爾瀏覽儀表板的存取次數，總請求數頂多數百次，僅佔每日 100,000 次免費額度的 **$< 0.5\%$**。

### 2. D1 讀取筆數 (Row Reads) 分析
- 每次檢查排程會讀取 1 次 `sites` 清單 (3筆) 及查詢各站點的最後一筆狀態日誌 (3筆)，共約 6 筆。
- 6 筆 $\times$ 每天 288 次 = **約 1,728 筆 / 天**。
- **結論**：僅佔每日 5,000,000 筆免費額度的 **$0.03\%$**。

### 3. D1 寫入筆數 (Row Writes) 分析
- **探測期間寫入**：每次探測寫入 `uptime_logs` 日誌 (3筆) 並更新 `sites` 表的最後檢查時間 (3筆)。
  - 每天探測寫入：6 筆 $\times$ 288 次 = 1,728 筆。
- **清理排程寫入 (DELETE 也算寫入資源)**：每天排程刪除大於 90 天前的舊資料，因為每天產生約 864 筆，所以大概每天也會刪除 864 筆。
- **報警紀錄寫入**：偶爾發生的故障報警寫入（次數極少）。
- 總寫入：約 **2,600 筆 / 天**。
- **結論**：消耗量極低，僅佔每天 100,000 筆免費額度的 **$2.6\%$**。如果未來增加到 30 個受監控網站，每天也只會消耗約 26,000 筆，仍在安全範圍內。

### 4. D1 儲存空間 (Storage) 分析
- 單筆 `uptime_logs` 的紀錄推算大小約 100 Bytes。
- 每天每個站點會產生 288 筆紀錄 (3 個站點共產生 864 筆)。
- 保留 90 天的最大累積量：864 $\times$ 90 = 77,760 筆。推估佔用約 **7 MB ~ 10 MB** 空間。
- **結論**：與 5 GB (5,120 MB) 的額度相比微乎其微，僅佔用約 **$0.15\%$** 的存空間。

## 綜合結論

HihiMonitor 使用的架構**極為輕量與節省資源**。以單一 Cloudflare 帳密的免費額度，在維持 5 分鐘探測頻率的條件下，系統完全可以輕鬆同時監管高達 **30 ~ 50 個網站**，從計算力到資料寫入限制都相當充裕，是一組可永久免費營運的理想解決方案。
